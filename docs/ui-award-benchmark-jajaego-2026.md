# 자재GO UI 디자인 어워드 벤치마크

- 작성일: 2026-07-24
- 적용 대상: 자재GO, 타일GO, 바스GO, 샘플GO, 관리자 운영 화면
- 목적: 국제 UI/UX 어워드 수상작과 우수 사례를 분석하고, 자재GO의 실제 검색·구매·운영 흐름에 적용할 디자인 기준을 정한다.

## 1. 결론

자재GO가 따라가야 할 방향은 화려한 전시형 웹사이트가 아니라 **검색과 주문이 빠른 B2B 커머스 UI**다.

어워드 수상작에서 가져올 핵심은 다음 다섯 가지다.

1. 복잡한 업무를 한 화면에서 이해할 수 있게 단순화한다.
2. 상품 이미지와 핵심 정보의 시각적 우선순위를 명확하게 만든다.
3. 사용자의 다음 행동을 한 가지씩 선명하게 제시한다.
4. AI 결과에는 판단 근거와 수정 가능한 조건을 함께 보여준다.
5. 모바일, 저속 네트워크, 접근성까지 디자인 품질에 포함한다.

반대로 과도한 3D, 긴 인트로, 스크롤 강제 효과, 숨겨진 메뉴, 의미 없는 모션은 자재GO에 적용하지 않는다. 이러한 표현은 브랜드 전시 사이트에는 효과적일 수 있지만, 반복 검색과 현장 주문이 중심인 자재GO에서는 속도와 신뢰를 떨어뜨린다.

## 2. 조사 기준

### iF Design Award

iF는 디자인을 `Idea`, `Form`, `Function`, `Differentiation`, `Sustainability`의 다섯 기준으로 평가한다. Function에는 사용성, 효율, 사용자 이익이 포함되고 Differentiation에는 혁신성과 브랜드 적합성이 포함된다.

자재GO 적용 해석:

- 예쁘기만 한 화면은 부족하다.
- 검색 시간이 줄고 주문 오류가 감소해야 한다.
- 타일·욕실자재 업종에 맞는 고유한 탐색 경험이 있어야 한다.
- 모바일 접근성, 정보 보호, 불필요한 작업 감소도 품질에 포함한다.

출처: [iF Design Award 심사 기준](https://ifdesign.com/en/faqs), [iF Design Award 2025 심사 결과](https://ifdesign.com/en/if-magazine/newsroom/if-design-award-2025-the-winners)

### UX Design Awards

UX Design Awards는 사용자 조사에 근거한 관련성, 사용자 역량 강화, 혁신, 사업 가치와 책임, 전체적인 사용자 중심 사고, 디자인과 경험 품질을 평가한다. 특히 불필요한 기능을 줄이고, AI의 작동 방식과 데이터 사용을 투명하게 전달하며, 사용자 신뢰를 확보할 것을 강조한다.

자재GO 적용 해석:

- AI 검색은 “AI가 찾았습니다”에서 끝나면 안 된다.
- 사용자가 선택한 사이즈·마감과 이미지에서 해석한 색상·스타일을 보여줘야 한다.
- 검색 결과를 사용자가 다시 수정할 수 있어야 한다.
- 고객용 화면에서 내부 브랜드, 공급처, 원가가 노출되지 않아야 한다.

출처: [UX Design Awards 심사 기준](https://ux-design-awards.com/jury/judging-criteria)

### Red Dot

Red Dot의 디지털 수상 사례에서 반복되는 특징은 명확한 시각적 위계, 적은 색상, 콘텐츠 집중, 직관적인 조작, 접근성, 모바일 최적화다.

자재GO 적용 해석:

- 상품 이미지, 품명, 품번, 사이즈, 마감의 순서를 고정한다.
- 고객이 보는 화면에는 내부 운영 정보를 섞지 않는다.
- 검색과 장바구니 행동을 다른 장식 요소보다 우선한다.

### Awwwards

Awwwards의 전자상거래 사례는 강한 첫인상, 제품 중심 사진, 브랜드 스토리, 세밀한 모션에서 참고 가치가 있다. 다만 자재GO에는 전면적으로 적용하지 않고 상품 상세, 시공 사례, 프로모션 영역에 제한적으로 활용한다.

출처: [Awwwards 전자상거래 우수 사례 모음](https://www.awwwards.com/websites/e-commerce/)

## 3. 핵심 수상·우수 사례

### 3.1 Shikhar App by Unilever

- 구분: Red Dot 수상작, B2B 리테일 주문 앱
- 문제: 소매점이 영업사원에게 의존해 도매 상품을 주문해야 했다.
- 해결: 상품 주문, 주문 추적, 반품을 하나의 앱에서 처리했다.
- 강점: 데이터 사용량이 적고 네트워크가 약한 환경에서도 사용할 수 있도록 설계했다.

자재GO 적용:

- 현장에서도 검색·장바구니·주문 상태를 빠르게 확인하게 한다.
- 상품 카드에 불필요한 설명을 줄이고 핵심 필드만 유지한다.
- 로딩 중에는 빈 화면이 아니라 스켈레톤과 진행 상태를 보여준다.
- 네트워크 실패 시 재시도 버튼과 마지막 요청 상태를 남긴다.

출처: [Shikhar App by Unilever](https://www.red-dot.org/project/shikhar-app-by-unilever-66741)

### 3.2 Uno

- 구분: UX Design Awards 2025 선정작, AWS 재무 업무 플랫폼
- 문제: 20개가 넘는 도구와 데이터가 분산되어 있었다.
- 해결: 27개 도구를 하나의 맞춤형 대시보드로 통합했다.
- 성과: 보고 작업을 10시간에서 1시간으로 줄였고 데이터 정확도를 99%까지 높였다고 발표했다.
- AI 사용: 자연어 Text-to-SQL 질의로 비전문가의 데이터 접근을 단순화했다.
- 접근성: AA 접근성 기준을 반영했다.

자재GO 적용:

- 관리자 대시보드는 상품, 회원 승인, 주문, 품질, 실행 요청을 한 구조 안에서 연결한다.
- 역할에 따라 대표, 관리자, 운영 담당자가 먼저 보는 지표를 다르게 구성한다.
- 자연어 검색 결과는 데이터 필터로 변환된 내용을 함께 보여준다.
- 대시보드의 목적은 많은 숫자를 보여주는 것이 아니라 다음 조치를 빠르게 결정하게 하는 것이다.

출처: [UX Design Awards - Uno](https://ux-design-awards.com/winners/2025-1-uno)

### 3.3 SiftedAI

- 구분: UX Design Awards 2025 선정작, 물류 분석 B2B SaaS
- 문제: 풍부한 데이터와 AI 기능이 있었지만 정보가 잘 발견되지 않았고 사용자의 인지 부담이 컸다.
- 해결: 중요 점수를 중심에 두고 비교와 모델링 경로를 단순화한 통합 대시보드를 만들었다.
- 강점: 강력한 기능을 숨기지 않으면서도 사용자가 무엇을 해야 하는지 안내했다.

자재GO 적용:

- 검색 결과 상단에는 결과 수, 적용 조건, 정렬 기준만 우선 표시한다.
- 품질 대시보드는 “미확인 수”만 표시하지 않고 바로 수정할 목록으로 연결한다.
- 재고·주문·회원 승인에는 정상, 주의, 조치 필요의 세 단계 상태를 사용한다.

출처: [UX Design Awards - SiftedAI](https://ux-design-awards.com/winners/2025-1-siftedai-a-logistics-intelligence-software)

### 3.4 HP Workforce Experience Platform

- 구분: UX Design Awards 2025 선정작, 엔터프라이즈 관리 플랫폼
- 문제: 여러 제품이 서로 다른 UI와 사용 방식을 가지고 있었다.
- 해결: 공통 디자인 라이브러리와 반응형 구조로 경험을 통합했다.
- 규모: 78만 대 이상의 장치와 월 1,100만 건의 문제를 관리하는 사례로 소개되었다.
- 강점: 반복 가능한 공통 패턴이 기능 확장 속도를 높였다.

자재GO 적용:

- 타일GO, 바스GO, 샘플GO는 같은 버튼, 검색창, 카드, 상태 표시 규칙을 공유한다.
- 상품 종류에 따라 데이터는 달라도 UI 문법은 같아야 한다.
- 공통 컴포넌트를 먼저 만든 뒤 각 서비스의 카테고리만 다르게 구성한다.

출처: [UX Design Awards - HP Workforce Experience Platform](https://ux-design-awards.com/winners/2025-2-hp-workforce-experience-platform)

### 3.5 AI Powered Investing

- 구분: UX Design Awards 2025 AI 콘셉트
- 문제: 복잡한 실시간 데이터와 AI 점수가 사용자를 혼란스럽게 했다.
- 해결: 자연어 요약, 단순화된 점수 카드, 단계적 상세 공개를 적용했다.
- 교훈: 시각적으로 화려한 데이터 대시보드보다 판단 이유를 쉬운 말로 설명하는 것이 신뢰를 높였다.

자재GO 적용:

- AI 검색 결과에 `사이즈 일치`, `마감 일치`, `색상 유사`, `스타일 유사`를 분리해 보여준다.
- 유사도 숫자만 보여주지 말고 “왜 비슷한지”를 짧게 설명한다.
- 상세 분석은 펼쳐보기로 제공하고 기본 화면은 간결하게 유지한다.

출처: [UX Design Awards - AI Powered Investing](https://ux-design-awards.com/winners/2025-2-ai-powered-investing)

### 3.6 CIARTIC Move

- 구분: UX Design Awards 2025 Gold
- 강점: 색상으로 연결된 시각 안내, 정밀한 조작, 기억 기능, 현장 전문가 피드백을 반영했다.
- 교훈: 중요한 업무에서는 아름다움보다 오류 감소와 상태 인지가 우선이다.

자재GO 적용:

- 주문 상태, 승인 상태, 오류 상태는 같은 색상 규칙을 전 화면에서 유지한다.
- 위험한 작업은 확인 단계를 제공하고 완료 결과를 명확히 표시한다.
- 운영자가 반복 입력하는 값은 최근 값과 기본값을 안전하게 활용한다.

출처: [UX Design Awards - CIARTIC Move](https://ux-design-awards.com/winners/2025-1-ciartic-move-self-driving-3d-c-arm)

### 3.7 29CM

- 구분: Red Dot 수상작, 모바일 커머스
- 강점: 최소한의 색상과 선을 사용해 복잡성을 줄이고 상품 콘텐츠에 집중하게 했다.
- 콘텐츠 전략: 상품 판매만이 아니라 브랜드 이야기와 기획 콘텐츠를 분리해 제공했다.

자재GO 적용:

- 상품 목록은 이미지와 필수 정보만 보여준다.
- 시공 사례와 자재 설명 콘텐츠는 상품 카드 안에 섞지 않고 별도 영역으로 분리한다.
- 컬러는 네이비, 블루, 화이트, 중립 회색을 중심으로 제한한다.

출처: [Red Dot - 29CM](https://www.red-dot.org/project/29cm-21429)

### 3.8 Energy Plus

- 구분: Red Dot 2025 수상작
- 강점: 명확한 시각적 위계, 고대비 컬러, WCAG 준수 컴포넌트, 직관적인 정보 구조다.

자재GO 적용:

- 텍스트 대비는 WCAG AA 기준을 따른다.
- 선택 상태는 색상만으로 구분하지 않고 체크, 테두리, 라벨을 함께 사용한다.
- 키보드 포커스가 하단 고정 메뉴나 팝업에 가려지지 않게 한다.

출처: [Red Dot - Energy Plus](https://www.red-dot.org/ko/project/energy-plus-85165)

### 3.9 A-RT.COM

- 구분: Red Dot 수상작, 통합 쇼핑 플랫폼
- 강점: 고정된 메뉴 형태에 얽매이지 않고 모바일 화면의 콘텐츠 흐름에 맞춰 정보 구조를 조절했다.

자재GO 적용:

- 데스크톱의 왼쪽 필터는 모바일에서 하단 시트로 전환한다.
- 모바일에서는 필터를 항상 펼치지 않고 선택 개수와 핵심 조건만 고정 표시한다.
- 동일 기능을 화면 크기만 줄여 배치하지 말고 모바일 작업 순서에 맞게 재구성한다.

출처: [Red Dot - A-RT.COM](https://www.red-dot.org/de/project/a-rtcom-55159)

### 3.10 Shinsegae Department Store

- 강점: 많은 텍스트 대신 여백과 고품질 이미지를 사용하고 개인 맞춤 큐레이션을 제공했다.

자재GO 적용:

- AI 추천은 전체 상품을 무작위로 보여주는 대신 사용자의 검색, 이미지, 장바구니 조건을 반영한다.
- 추천 이유와 기준을 노출한다.
- 제품 이미지의 비율과 품질을 통일한다.

출처: [Red Dot - Shinsegae Department Store](https://www.red-dot.org/de/project/shinsegae-department-store-16526-16526)

### 3.11 MAZEDROP

- 구분: Red Dot 수상작, 전자상거래 플랫폼
- 강점: 최소한의 명확한 레이아웃으로 콘텐츠에 집중시키고 발견형 탐색을 제공했다.

자재GO 적용:

- 메인 화면은 기능 설명보다 검색 진입과 대표 카테고리를 우선한다.
- “최근 본 상품”, “비슷한 타일”, “현장 조건에 맞는 상품”을 발견형 영역으로 제공한다.
- 발견형 영역도 구매 흐름을 방해하지 않는 범위로 제한한다.

출처: [Red Dot - MAZEDROP](https://www.red-dot.org/de/project/mazedrop-49339)

## 4. 자재GO 디자인 원칙

### 원칙 1. 상품 찾기가 첫 화면의 주인공이다

고객이 가장 먼저 해야 할 행동은 상품 검색 또는 카테고리 선택이다. 긴 소개문, 큰 운영 설명, 관리자용 문구는 고객 화면에 노출하지 않는다.

메인 우선순위:

1. 통합 검색
2. AI 타일 검색
3. 타일GO
4. 바스GO
5. 샘플GO
6. 최근 본 상품과 장바구니

### 원칙 2. 고객용과 관리자용을 시각적으로도 분리한다

고객용:

- 내부 브랜드 비노출
- 공급처 비노출
- 원가 비노출
- 품질·마진 등급 비노출
- 상품과 구매 행동 중심

관리자용:

- 내부 브랜드를 첫 번째 필터로 제공
- 공급처, 원가, 품질 상태, 데이터 누락 상태 제공
- 승인, 주문, 품질 수정, 실행 요청 중심

### 원칙 3. 상품 카드의 정보 순서를 고정한다

고객용 상품 카드 기본 구성:

1. 대표 이미지
2. 품명
3. 품번
4. 사이즈
5. 마감
6. 장바구니 담기

상품 목록에서는 재고, 긴 설명, 공급처, 내부 브랜드를 표시하지 않는다. 원산지는 상세 페이지와 검색 이후 선택형 옵션에서만 사용한다.

### 원칙 4. 필터는 상품군마다 다르되 UI 문법은 같게 한다

타일GO 필터 순서:

1. 사이즈
2. 용도: 벽 / 바닥
3. 마감
4. 스타일
5. 색상

검색 결과 이후 보조 옵션:

- 원산지
- 두께
- 가격
- 재고 상태

바스GO 주요 카테고리:

- 수전
- 세면대
- 욕조
- 양변기
- 욕실장
- 거울
- 욕실 액세서리

샘플GO:

- 일반 상품 목록과 혼동되지 않게 샘플 주문 목적을 분명히 표시

### 원칙 5. AI는 전문가의 판단 과정을 보여준다

AI 타일 검색 입력:

- 자연어
- 현장 또는 타일 이미지
- 사이즈
- 마감

AI 검색 결과:

- 선택한 사이즈와 마감은 절대 필터
- 이미지의 색상, 패턴, 베인, 모양, 조각 구성을 분석
- 최대 50개 결과
- 비슷한 이유를 짧게 표시
- 사용자가 색상·마감·사이즈·스타일을 다시 조정 가능

권장 결과 설명:

> 600x600 무광 조건이 일치합니다. 웜베이지 색상과 잔잔한 스톤 패턴이 유사합니다.

피해야 할 결과 설명:

> AI 유사도 87

### 원칙 6. 한 화면에는 하나의 주 행동을 둔다

- 상품 카드: 담기
- 상세 페이지: 장바구니 담기
- 장바구니: 제안서 또는 실사 보정 진입
- 회원 승인: 승인 또는 반려
- 주문 상세: 상태 변경

동일한 강조 수준의 버튼을 여러 개 두지 않는다. 보조 행동은 테두리 버튼이나 메뉴로 낮춘다.

### 원칙 7. 상태와 오류를 숨기지 않는다

필수 상태:

- 로딩
- 데이터 없음
- 검색 결과 없음
- 네트워크 오류
- 권한 없음
- 저장 중
- 저장 완료
- 부분 실패

오류 문구는 기술 코드를 그대로 노출하지 않고, 원인과 다음 행동을 알려준다.

예:

> 상품을 불러오지 못했습니다. 연결 상태를 확인한 뒤 다시 시도해 주세요.

## 5. 화면별 상세 설계 기준

### 메인

- 첫 화면에 검색창과 핵심 서비스 진입을 배치한다.
- 브랜드의 성격은 색상, 로고, 정돈된 사진으로 표현한다.
- 기능을 설명하는 긴 문단은 넣지 않는다.
- 다음 섹션이 첫 화면 아래에 일부 보이게 구성한다.

### 통합 상품 검색

- 자연어와 이미지 검색을 한 페이지에서 제공한다.
- 입력 후 해석된 조건을 칩으로 표시한다.
- 검색 전에는 입력에 집중하고, 검색 후에는 결과와 필터에 집중한다.
- 검색 결과를 유지한 채 필터를 즉시 적용한다.

### 타일GO 상품 목록

- 모바일 2열, 데스크톱 4열을 기본으로 한다.
- 이미지 영역의 비율을 통일한다.
- 긴 품명은 두 줄까지 표시하고 이후 생략한다.
- 카드 높이가 콘텐츠에 따라 흔들리지 않게 고정 영역을 사용한다.
- 페이지 전환은 버튼과 스와이프를 모두 지원하되 현재 페이지 위치를 유지한다.

### 바스GO 상품 목록

- 타일GO와 분리된 상품 페이지를 사용한다.
- 대분류와 세부분류를 네이버 스마트스토어처럼 예측 가능한 방식으로 제공한다.
- 카드의 담기 버튼은 타일GO와 같은 작은 크기로 통일한다.
- 상품 상세로 진입할 수 있어야 한다.

### 상품 상세

- 이미지가 가장 먼저 보이게 한다.
- 품명, 품번, 사이즈, 마감, 원산지, 두께를 명확하게 구분한다.
- 고객 권한에 따라 가격을 노출한다.
- 관리자만 내부 브랜드, 공급처, 원가를 볼 수 있다.
- 추가 이미지는 클릭해 확대할 수 있게 한다.

### 장바구니

- 상품별 면적 또는 낱장 입력과 박스 수 계산을 제공한다.
- 프로 제안서와 실사 보정은 서로 다른 목적의 행동으로 유지한다.
- 실사 보정 진입 시 장바구니의 타일 후보를 작은 이미지, 모델명, 사이즈로 보여준다.

### 실사 보정

- 공간 이미지를 분석해 거실, 욕실, 주방, 외벽 등의 공간 유형을 제안한다.
- 원하는 인테리어 스타일을 선택하게 한다.
- 실행 중에는 중앙 로딩 그래픽과 단계별 진행 상태를 표시한다.
- 결과는 원본과 보정 이미지를 슬라이더로 비교한다.
- 보정 결과는 그래픽 렌더가 아니라 전문 사진사가 촬영한 인테리어 사진의 질감을 목표로 한다.

### 마이페이지

- 현재 주문
- 지난 주문
- 장바구니
- 회원 등급
- 사업자 인증 상태
- 담당자 정보
- 거래처 관리
- 반품 주문

가장 자주 확인하는 주문과 인증 상태를 위에 둔다.

### 관리자 대시보드

- 오늘 처리해야 할 회원 승인, 주문 접수, 오류, 데이터 품질 작업을 먼저 보여준다.
- 통계 카드보다 실행 가능한 작업 목록을 우선한다.
- 브랜드 필터는 내부 관리자에게만 첫 번째 필터로 제공한다.
- 실행 요청은 접수, 처리 중, 완료, 실패 상태와 결과 메모를 남긴다.

## 11. 어워드 출품 관점의 자재GO 차별점

자재GO가 디자인 어워드에서 경쟁력을 가지려면 단순히 화면을 예쁘게 만드는 것보다 다음의 실제 성과를 입증해야 한다.

1. 전화와 거래처별 사이트 확인에 걸리던 상품 탐색 시간을 얼마나 줄였는가.
2. 자연어와 이미지 검색이 전문가의 상품 탐색 방식을 얼마나 잘 재현하는가.
3. 고객에게는 단순하지만 내부 운영자는 방대한 상품을 정확히 관리할 수 있는가.
4. 현장 사진과 장바구니 상품을 연결해 구매 결정을 얼마나 쉽게 만들었는가.
5. 저속 네트워크와 모바일 현장 환경에서도 핵심 작업을 완료할 수 있는가.
6. AI가 추천한 이유를 사용자가 이해하고 통제할 수 있는가.
7. 사용자 조사와 실제 검색 로그를 통해 반복 개선했는가.

가장 적합한 출품 방향은 `B2B Marketplace`, `Enterprise Solution`, `AI Search Experience`, `Retail & Distribution`, `Customer Portal` 영역이다.

- [ ] 로딩과 실패 상태가 분명한가?
- [ ] 오류 후 사용자가 다음에 무엇을 해야 하는지 알 수 있는가?
- [ ] 스크롤이나 모션이 상품 비교를 방해하지 않는가?
- [ ] 검색 조건과 결과의 관계를 이해할 수 있는가?
- [ ] 키보드만으로 주요 작업을 수행할 수 있는가?
- [ ] 화면 전체를 새로고침하지 않고 필터와 페이지가 자연스럽게 전환되는가?
- [ ] 작은 화면에서도 하단 메뉴가 콘텐츠를 가리지 않는가?
- [ ] 관리자 화면은 통계보다 처리할 업무를 우선하는가?

# Product DB Rebuild Spec

새 상품 DB 정리 작업 기준.

## 수집 방식

사용자가 거래처/제조사/공급사 링크를 제공하면, 해당 사이트에 로그인해서 상품 목록과 상세 페이지를 확인한다.
로그인 정보는 `.env`에만 저장하고 코드, 문서, DB 백업, Git 커밋에는 포함하지 않는다.

## 저장 필드

새 DB 상품은 아래 필드를 기준으로 정리한다. 원 사이트에 해당 내용이 없으면 공란으로 둔다.

| 표시명 | 내부 키 | 설명 |
| --- | --- | --- |
| 사이즈 | `size` | 예: 600x600, 300x600, 1200x2780 |
| 모델명 | `modelName` | 사이트의 품명, 모델명, 제품명과 같은 의미 |
| 소재 | `material` | 자기질, 도기질, 포세린, 세라믹, 스톤 등 |
| 표면 | `surface` | 무광, 유광, 논슬립, 러프, 폴리싱 등 |
| 패턴 카테고리 | `patternCategory` | 스톤, 마블, 시멘트, 솔리드, 테라조, 우드, 패턴, 기타 |
| 제조국 | `countryOfOrigin` | 원산지, 제조국, 생산국과 같은 의미 |
| pcs/box | `pcsPerBox` | 박스당 장수 |
| m2/box | `sqmPerBox` | 박스당 면적 |
| 색상 | `color` | 화이트, 그레이, 베이지, 마블 등 |
| 특징 | `features` | 방수, 논슬립, 포세린, 빅슬랩 등 부가 설명 |
| 재고 | `stockQty` | 숫자로 정리 가능한 총 재고 수량 |
| 재고 원문 | `stockText` | 사이트에 표시된 원문 재고. 예: 파주창고(88), 하남창고(300 이상) |
| 원가 | `costPrice` | 거래처 사이트에 표시되는 기본 금액. 예: 도매가, 공급가, 단가 |
| A등급금액 | `gradeAPrice` | A등급 판매/공급 금액 |
| B등급금액 | `gradeBPrice` | B등급 판매/공급 금액 |
| C등급금액 | `gradeCPrice` | C등급 판매/공급 금액 |

## 보조 필드

상품 추적과 재고 동기화를 위해 아래 필드는 가능하면 같이 저장한다.

| 표시명 | 내부 키 | 설명 |
| --- | --- | --- |
| 원본 사이트 | `sourceSite` | 예: tile114 |
| 원본 URL | `sourceUrl` | 상품 상세 URL |
| 원본 상품 ID | `sourceProductId` | 거래처 상품 번호 또는 상세 페이지 ID |
| 이미지 | `image` | 대표 이미지 |
| 추가 이미지 | `imageUrls` | 상세/확대/시공 이미지 목록 |
| 마지막 확인 | `lastSyncedAt` | 재고/가격 마지막 확인 시각 |

## 매핑 규칙

- 사이트에 같은 이름의 필드가 없어도 같은 의미이면 해당 DB 필드에 입력한다.
- `품명`, `제품명`, `상품명`, `모델`, `모델명`은 `modelName`으로 본다.
- `규격`, `사이즈`, `크기`는 `size`로 본다.
- `재질`, `소재`, `타입`이 타일 재질 의미이면 `material`로 본다.
- `마감`, `표면`, `광택`, `논슬립`은 `surface`로 본다.
- `스톤`, `마블`, `시멘트`, `솔리드`, `테라조`, `우드`, `패턴`처럼 시각적 무늬 분류는 `patternCategory`로 본다.
- `단위`에 `BOX/들이(32)/0.74㎡/14.2KG`처럼 적혀 있으면 `pcsPerBox=32`, `sqmPerBox=0.74`처럼 분리한다.
- `재고량`, `재고`, `창고`, `보유수량`은 재고 정보로 본다.
- 재고가 `파주창고(88) 하남창고(300 이상)`처럼 창고별로 표시되면 전체 문구는 `stockText`에 저장하고, 숫자로 확정 가능한 합계는 `stockQty`에 저장한다.
- `300 이상`처럼 정확한 수량이 아닌 값은 `stockText`에는 그대로 보존하고, `stockQty`에는 최소 확인 수량 기준으로 넣거나 대량 수집 전 사용자에게 기준을 확인한다.
- 거래처 사이트에 보이는 `도매가`, `공급가`, `단가`, `매입가` 등 기본 금액은 원가로 보고 `costPrice`에 저장한다.
- `A등급`, `B등급`, `C등급`처럼 등급별 판매 금액이 명확히 따로 있을 때만 `gradeAPrice`, `gradeBPrice`, `gradeCPrice`에 저장한다.
- 사이트에 가격이 하나만 있고 등급 구분이 없으면 `costPrice`에 넣고, `gradeAPrice`, `gradeBPrice`, `gradeCPrice`는 공란으로 둔다.
- 원본에 없는 값은 추측해서 채우지 않는다. 단, 상품명/설명에서 명확히 추출 가능한 값은 입력한다.

## 작업 원칙

- 기존 상품 DB는 백업을 유지한다.
- 새 DB 작업 중에는 상품페이지에서 기존 상품을 숨길 수 있다.
- 새 상품은 먼저 샘플 수집으로 필드 매핑을 검증한 뒤 대량 반영한다.
- 대량 반영 전에는 수집 결과 JSON/CSV를 별도 파일로 남겨 검토한다.
